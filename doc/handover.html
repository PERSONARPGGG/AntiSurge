🛠 환경 설정

필요 도구

  • Node.js (validate.js, split-modules.js 실행용)
  • Firebase CLI (npm install -g firebase-tools)
  • Git + .githooks/pre-commit 자동 검증 훅

Git Hooks 설정

# 저장소 클론 후 1회만 실행
git config core.hooksPath .githooks

Firebase 배포 (hosting only)

firebase deploy --only hosting --token "YOUR_TOKEN"
⚠️
Firebase 토큰과 GitHub 토큰은 절대 코드에 포함하지 말 것. .env나 별도 파일로 관리.

로컬 테스트

파일을 직접 file://로 열면 됩니다 (ES module이 아닌 일반 script 태그 사용). 또는:

npx serve .  # 로컬 HTTP 서버 (선택사항)

🔄 개발 워크플로

표준 사이클

  1. 해당 src/*.js 파일 수정
  2. 브라우저에서 테스트
  3. node validate.js 실행 (또는 커밋 시 자동 실행)
  4. git add src/[수정파일] → git commit (훅이 자동 검증)
  5. Firebase 배포

모듈 경계 조정 시

# split-modules.js 의 라인 범위 수정 후:
node split-modules.js
node validate.js
⚠️
split-modules.js는 game.js.bak을 원본으로 읽어 src/ 파일들을 덮어씁니다. 개별 수정한 src/*.js가 있으면 먼저 백업.

validate.js 검사 항목 (18개)

  • 인덱스 루프 내 직접 사망 호출 (takeDamage / die)
  • 전역 배열 재할당 (= [] 패턴)
  • for..of 루프 중 원본 배열 splice
  • 무기 update() level>0 가드
  • STATE 전환 유효성
  • 핵심 모달 닫기 후 게임루프 재시작
  • activeBoss null 체크
  • HTML ID 참조 무결성
  • 각 src/*.js 문법 (node --check)
  • setTimeout 내 player null 체크
  • mines/blackHoles 루프 경계 체크
  • 파일 크기
  • TODO/FIXME 주석

🔀 게임 상태 머신

STATE_MENU → STATE_PLAYING

STATE_PLAYING ↔ 아래 상태들과 상호 전환:

STATE_LEVEL_UP STATE_STAGE_CLEAR STATE_STAGE_BONUS STATE_SHOP STATE_CURSE STATE_PAUSED
STATE_PLAYING → STATE_GAME_OVER
🚨
상태 전환 후 반드시 ensureGameLoopRunning() 호출. 미호출 시 게임 루프가 멈춥니다.

🛡 버그 방지 체크리스트

과거 발생한 버그 패턴. 코드 수정 시 반드시 확인.

  • 1
    상태 전환 후 루프 재시작 모달 닫기 / 상태 복원 후 ensureGameLoopRunning() 호출 필수
  • 2
    pendingLevelUps 카운터 레벨업 모달 표시 중 추가 레벨업 → pendingLevelUps++ 후 큐 처리
  • 3
    전역 배열 초기화 enemies = [] 금지 → enemies.length = 0 사용
  • 4
    루프 중 스테이지 클리어 유발 enemies 루프 중 triggerStageClear() 유발 코드 → .length = 0 + 경계 체크
  • 5
    isStageClearAnim 플래그 클리어 연출 중 상태 전환 → isStageClearAnim 체크 후 결정
  • 6
    pendingBossCurse 중복 방지 보스 처치 시 저주 1회만 → pendingBossCurse 플래그 확인
  • 7
    보스 스테이지 전환 순서 보스 스폰 직전 gameState = STATE_PLAYING 먼저 설정 후 스폰
