01. 스택 철학

🧬

빌드 없는 브라우저 네이티브

Node.js 빌드 파이프라인, 번들러, 트랜스파일러 없이 브라우저가 직접 실행하는 구조.
초기에는 단일 game.js (~8000줄)였으나, 유지보수성을 위해 src/ 15개 모듈로 분리.
분리 후에도 전역 스코프를 공유하는 <script> 태그 방식이므로 빌드 과정 없이 Firebase Hosting에 즉시 배포 가능.

06. 성능 최적화

  • Canvas offscreen 캐싱 — 배경 그리드/별 효과를 OffscreenCanvas에 1회 렌더 후 재사용
  • shadowBlur 제거 — 렌더 루프에서 blur 연산 제거로 GPU 부하 감소
  • MAX_PROJECTILES 제한 — 투사체 상한선으로 과부하 방지
  • requestAnimationFrame — vsync 기반 루프, 탭 비활성 시 자동 일시정지
  • dt 클램핑 — dt > 100ms면 16.66ms로 고정 (탭 복귀 스파이크 방지)
  • 배열 순회 최적화 — 루프 중 splice 대신 인덱스 역순 탐색
  • touch 이벤트 passive — { passive: true } 옵션으로 스크롤 블로킹 방지
  • AudioContext 지연 초기화 — 첫 사용자 상호작용 후 생성 (브라우저 정책 대응)
