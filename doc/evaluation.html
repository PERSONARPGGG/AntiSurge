🎮 게임 평론가 시점 CRITIC · 6.5 / 10
"안티바이러스 컨셉은 선명하고 독창적이다. 시스템 완성도는 상업 게임 수준에 가깝지만, 3판 이후 새로운 걸 보여주지 못한다는 점이 치명적 약점이다."
▲ 강점
  • 테마 차별성 — 루트킷·방화벽·패치봇 등 용어 통일로 사이버펑크 세계관 몰입감 탁월
  • 즉각 진입 — 설치 없이 URL 하나로 플레이, 클래스 미리보기로 진입 장벽 최소화
  • 멀티플레이 — 웹 브라우저 기반 실시간 협동/경쟁은 이 장르에서 극히 드물다
  • 감염 협약 시스템 — 보스 처치 후 리스크/리워드 선택 구조, 긴장감 있는 결정 순간 제공
  • 메타 루프 — 데이터 코어 → 영구 업그레이드 → 다음 런 동기 부여 기본 구조 완성
▼ 약점
  • 콘텐츠 밀도 부족 — 적 4종 vs 뱀파이어 서바이버스 50+종. 10판 후 새로운 게 없음
  • 온보딩 부재 — 감염 협약, 메타 업그레이드, 멀티 시스템 설명이 전혀 없다
  • 보스 단조로움 — "더 크고 빠른 원"에 불과. 패턴·페이즈 없는 보스는 긴장감을 만들지 못함
  • 타격감 약함 — SFX 피드백이 얕아 도파민 루프가 약하다. 킬 쾌감이 떨어짐
  • 스테이지 100 이후 공허 — 무한 모드에 목표·보상·변화가 없다

▲ 기술적 성취
  • HTML + CSS + JS 단일 파일 스택으로 상업 게임 수준 기능 구현
  • Web Audio API 신스 BGM 4트랙 — 음원 파일 없이 실시간 사운드 생성
  • Firebase RTDB 실시간 협동/경쟁 멀티플레이 + BroadcastChannel 폴백
  • Offscreen Canvas 캐싱, shadowBlur 최적화로 60fps 유지
  • validate.js 13개 자동 검사 — pre-commit 훅으로 배포 전 버그 방지
  • Safe-area-inset 대응, 모바일 자동 전체화면, 터치 조이스틱 완성
△ 기술 부채 / 리스크
  • 단일 파일 한계 — 7400줄 돌파 후 기능 추가 시 충돌·디버그 비용 급증. 장기적으로 ES Modules 분리 검토 필요
  • Object Pooling 미적용 — enemies, projectiles, particles 매 프레임 생성/소멸 → 장시간 GC 히칭 발생 가능
  • Firebase 비용 구조 — 유저 100명+ 시 RTDB 읽기 비용 급증. Firestore + 클라이언트 캐싱 전환 미리 설계 권장
  • 세이브 마이그레이션 취약 — 버전업마다 세이브 구조 변경 시 유저 데이터 손실 위험. 버전 플래그 + 마이그레이션 함수 필요
  • 콘텐츠 추가 확장성 — 무기/적 데이터가 하드코딩. JSON 외부 데이터 분리 시 콘텐츠 추가 비용 대폭 감소

"오픈 소스 클론이 아닌, 자기 색깔을 가진 웹게임. 시스템은 상업 게임 수준에 가깝지만, 유저를 3판 이상 잡아둘 콘텐츠와 첫인상이 아직 부족하다. 무기 시너지 + 콘텐츠 해금 + 보스 패턴 3가지만 추가해도 6.5점 → 8점+ 도달 가능하다."
PRIORITY 01
무기 시너지 진화
2~3개 조합 진화 무기만 추가해도 빌드 다양성이 폭발적으로 증가. 뱀서 리텐션의 핵심.

PRIORITY 03

페이즈가 있는 보스 하나만 있어도 게임 인상이 완전히 달라진다. 클라이맥스 제공.
