7.8
/ 10 · RECOMMENDED
"히든 직업 체인이 게임의 성격을 완전히 바꿨다. v0.09는 '뱀서 클론'이었고, v0.10은 '파고드는 맛이 있는 게임'이다. 트루 엔딩의 흡수 역전 메커닉은 이 장르 웹게임 전체에서 가장 독창적인 보스 설계다. 단, 거기 도달하기까지의 여정이 여전히 단조롭다는 점이 치명적이다."
▲ 상세 강점
  • 흡수 역전 보스 설계 — HP가 0에 도달해도 죽지 않고, 충분한 피해를 입혀야만 역전이 트리거된다. 보스를 '이기는' 것이 아니라 '감염시키는' 아이디어가 게임 테마와 완벽히 일치한다. 이 수준의 개념적 일관성은 AA 게임에서도 드물다.
  • 히든 클래스 해금 체인 — 재머→크래커→글리치 댄서→패러사이트 4단계 체인은 단순한 잠금 해제가 아니라 탐색 서사를 만든다. 중간 단계를 달성할 때마다 '다음에 뭐가 나올까'라는 궁금증을 유지한다.
  • 업적 힌트 구조 — '더 깊이 들어가라', '문이 열릴 것 같다' 같은 암시적 텍스트는 스포일러 없이 FOMO를 자극하는 정석 설계다.
  • 패러사이트 클래스 아이덴티티 — 흡수로 강해지고, 흡수로 엔딩까지 가는 일관된 클래스 철학. 다른 어떤 클래스도 이 수준의 정체성을 갖지 못한다.
  • 설치 없는 즉시 플레이 — URL 하나로 시작하는 경험은 2026년 기준으로도 웹게임의 가장 큰 차별점이다.
▼ 상세 약점
  • 일반 보스의 치명적 단조로움 — 스테이지 10~90에 걸쳐 보스는 전부 '더 크고 빠른 원형 적'이다. 트루 엔딩 보스와의 격차가 너무 커서 오히려 일반 보스가 더 실망스럽게 느껴진다.
  • 온보딩 부재 — 첫 플레이에서 패시브 트리, 히든 클래스, 흡수 시스템, 감염 협약의 존재를 전혀 안내하지 않는다. 대부분의 신규 유저는 이 게임이 그냥 '평범한 뱀서 클론'이라고 오해하고 떠난다.
  • 비주얼 밀도 부족 — 킬 쾌감의 70%는 비주얼 피드백에서 온다. 현재 파티클 이펙트는 인상을 남기기에 부족하고, 도형 기반 적 디자인은 스크린샷 공유 욕구를 만들지 못한다.
  • 트루 엔딩 접근성 문제 — 50회 흡수 + 히든 3클래스 각각 100스테이지 클리어는 캐주얼 유저에게 현실적으로 불가능한 조건이다. 엔딩 힌트는 있지만 달성 경로가 너무 멀다.
  • 사운드 피드백 약함 — 신스 BGM은 창의적이나, 킬 SFX의 얕은 피드백이 타격감과 도파민 루프를 방해한다.

▲ 기술 성취
  • 모듈 아키텍처 — config → audio → player → weapons → projectiles → enemies → entities → stage → flow → loop → upgrades → endgame → finalstage → extras → input 순서의 의존성 그래프가 순환 없이 깔끔하다
  • _endingCutsceneIds 패턴 — setTimeout ID를 배열로 추적하고 스킵 시 전체 취소. 좀비 타이머 방지의 교과서적 구현
  • 다중 조건 해금 — requires(배열 재귀) + stats(배열 AND) 동시 지원하는 isClassUnlocked()는 확장 비용 없이 복잡한 조건을 처리
  • 파이널 보스 데이터 분리 — finalWaveVirusCore/finalWaveOrigin을 전역으로 두고 기존 충돌 시스템의 allTargets에만 추가하는 방식이 기존 코드 변경을 최소화
  • 자동 검증 파이프라인 — 리팩터링 후 모든 항목 통과. 솔로 개발의 안전망으로 이 수준의 자동화는 이례적
△ 기술 부채 / 위험 요소
  • Object Pooling 미적용 — 트루 엔딩 웨이브 1에서 80킬 + 수십 파티클 생성·소멸이 반복된다. 장시간 세션에서 GC 히칭 발생 가능. 적·투사체·파티클에 풀 도입이 권장됨
  • 세이브 버전 플래그 부재 — _hiddenClassClears·_parasiteEnding 등 v0.10 신규 필드는 구버전 세이브에서 undefined. 읽을 때 || {}로 방어하나, 마이그레이션 함수 없이 필드가 계속 늘어나면 언젠가 무너진다
  • Firebase 비용 구조 — 현재 BroadcastChannel 폴백 덕에 비용 0이지만, 실제 Firebase RTDB 활성화 시 100+ 동시 유저부터 비용 급증. Firestore + presence 분리 설계 권장
  • 모바일 파이널 스테이지 미검증 — VirusOriginBoss의 대규모 투사체 폭발 연출이 모바일 저사양 기기에서 프레임 드롭을 유발할 수 있음

게임 행동경제학 연구자 · 플레이어 동기·중독 메커닉 분석 전문
7.2
/ 10 · STRONG HOOKS
"흡수 역전 게이지가 95%에 도달했을 때 플레이어가 느끼는 긴장감은 행동경제학적 '거의 달성' 효과(Near-Miss Effect)의 교과서 사례다. 보스를 직접 죽이지 못하고 게이지를 채워야 한다는 구조는 몰입을 유지하면서 보상 시점을 지연시킨다 — 도파민 루프의 이상적인 설계."
▲ 심리학적 강점
  • Near-Miss Effect 활용 — 흡수 역전 게이지 80%+ 구간에서 시각 피드백이 강해지며 '거의 됐어!' 인식을 유발. 이 구간의 몰입도가 극대화된다
  • 히든 콘텐츠의 '신비감' — 존재를 암시하되 전부 보여주지 않는 업적 텍스트는 Zeigarnik Effect(미완성 과제 기억 효과)를 활용한다. 유저는 자꾸 생각하게 된다
  • 계층적 목표 구조 — 단기/중기/장기 목표의 공존은 어떤 플레이어 유형(캐주얼~하드코어)도 즉각적인 다음 목표를 갖게 해준다
  • 클래스 아이덴티티의 소유감 — '나는 패러사이트 유저'라는 정체성 부여는 이탈 저항을 높인다. 히든 클래스일수록 소유 욕구가 강해진다
▼ 심리학적 약점
  • 초반 도파민 루프 약함 — 처음 3분에 킬 피드백(SFX·파티클·콤보 카운터 감각)이 충분히 자극적이지 않다. 첫 킬의 쾌감이 약하면 '다음 킬' 동기가 생기지 않는다
  • 레벨업 카드의 랜덤성 부족 — 현재 업그레이드 풀이 예측 가능하다. 예상치 못한 선택지가 나타날 때 뇌가 더 활성화된다 (Variable Reward Schedule)
  • 트루 엔딩의 심리적 거리 — 조건이 너무 복잡하면 플레이어는 시작조차 하지 않는다 (Goal Distance Effect). 중간 마일스톤('흡수 25회', '히든 1클래스') 달성 시 축하 피드백이 필요하다
  • 보스 처치의 카타르시스 부재 — 일반 보스는 '점점 녹이다 죽는' 경험이다. 극적인 죽음 연출(폭발·빛·텍스트) 없이는 성취감이 축적되지 않는다
핵심 제안: 초반 3분에 "첫 킬 → 콤보 5 → 레벨업 → 첫 보스 처치"의 도파민 스파이크 4연속을 설계하라. 이 시퀀스가 10분 리텐션을 결정한다. 트루 엔딩은 이미 충분히 좋다 — 거기까지 데려가는 여정의 피드백이 문제다.

💡 기획 아이디어 제안 5가지 즉시 논의 가능
현재 시스템을 잘 활용하면서 큰 공수 없이 게임 경험을 크게 바꿀 수 있는 아이디어들입니다. 각 아이디어에 구현 규모와 예상 임팩트를 함께 표기했습니다.
IDEA 01
바이러스 변이 (적 돌연변이 시스템)
스테이지가 진행될수록 일반 적에게 랜덤 변이가 추가됩니다. 예: 방어막 바이러스(투사체 첫 타 무효), 분열 바이러스(죽으면 2마리로 분열), 자폭 바이러스(플레이어 근접 시 폭발), 재생 바이러스(천천히 HP 회복). 적 스폰 시 15% 확률로 변이 부여 — 각 변이는 작은 아이콘으로 적 위에 표시됩니다. 무기 선택이 전략적 의미를 갖게 됩니다. (자폭 적 → 원거리 무기 우선 등)
구현 규모: 중 (적 렌더·사망 로직 수정) 임팩트: ★★★★★ — 런 내 전략 깊이 폭발적 증가 기존 코드와 충돌 없음
IDEA 02
코드 크랙 (보스 취약점 타이밍 창)
보스 HP가 20% 이하일 때 3~5초의 취약점 창이 열립니다. 화면 중앙에 작은 사이버 코드 아이콘이 깜빡이고, 이 구간에 가하는 피해가 2.5배 적용됩니다. 취약점 창을 놓치면 보스가 일시적으로 방어막을 치고 이후 다시 창이 열립니다. 패러사이트는 이 구간에 흡수 시 게이지 3배 충전 — 클래스 시너지까지 연결됩니다. 구현: 타이머 + 비주얼 오버레이만 필요. 기존 Boss 클래스에 필드 몇 개 추가로 완성 가능.
구현 규모: 하 (보스 클래스에 타이머 추가) 임팩트: ★★★★☆ — 보스전 긴장감 대폭 상승 1~2시간 구현 가능
IDEA 03
네트워크 분기 (멀티 전용 동시 보스)
2인 협동 멀티플레이 시, 보스 스테이지에서 보스가 두 개의 분신으로 나뉩니다. 각 플레이어가 자신의 분신을 담당하며, 두 분신의 HP가 동시에 소진될 때만 사망 처리됩니다. 한 쪽 분신이 먼저 죽으면 합체되어 남은 HP 50%로 강화된 통합 형태가 됩니다. 솔로 플레이 시에는 기존 보스가 그대로 등장합니다. 멀티플레이의 유일한 전용 콘텐츠로 협동 유인이 극적으로 강해집니다.
구현 규모: 중 (멀티 상태 동기화 필요) 임팩트: ★★★★☆ — 멀티 플레이 가치 대폭 상승 기존 mpSetupChannel 활용
IDEA 04
백도어 루트 (조건부 스테이지 스킵 포털)
특정 조건을 달성하면 맵 한쪽에 임시 포털이 등장합니다. 예: '30초 내 20킬', '콤보 30 이상 유지', '피격 없이 1분 생존'. 포털에 닿으면 다음 스테이지 하나를 스킵하고 보너스 XP를 획득합니다. 단, 스킵한 스테이지의 드롭 아이템과 골드는 포기하게 됩니다. 리스크·리워드 선택 구조를 스테이지 이동에도 적용하는 아이디어입니다. 고수 플레이어에게는 스피드런 경로가 되고, 초보자에게는 '이런 게 있었어?' 발견의 기쁨이 됩니다.
구현 규모: 하 (FieldItem 클래스 확장) 임팩트: ★★★☆☆ — 고수·스피드런 층 유입 데일리 챌린지와 연계 가능
IDEA 05
클래스 스토리 코덱 (런 종료 후 스토리 해금)
각 클래스로 특정 조건(첫 클리어, 스테이지 50 돌파, 특수 업적 등)을 달성하면 1~3줄의 세계관 로그가 해금됩니다. 결과 화면에 "새 로그 해금됨" 알림이 뜨고, 메뉴의 클래스 선택 화면에서 해금된 로그를 열람할 수 있습니다. 예: 패러사이트 첫 클리어 → "바이러스를 흡수하는 자. 이것이 최선인지, 감염인지 — 아직 모른다." 트루 엔딩 달성 → 마지막 로그 해금 + 전체 스토리가 하나의 타임라인으로 연결됩니다. 세계관 몰입도가 폭발적으로 증가하며, 완료 욕구(컬렉션 심리)를 자극합니다.
구현 규모: 하 (텍스트 데이터 + 저장 플래그) 임팩트: ★★★★★ — 세계관 몰입 + 리플레이 동기 동시 해결 마케팅 자산으로도 활용 가능

  • 그러나 — 9.0~9.2점은 완전히 달성 가능하며, 이 점수대는 인디게임 최상위 10%에 해당한다. 현재 방향성이 옳다.
